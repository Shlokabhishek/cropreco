üéØ 1. System Overview

The Crop Recommender System is an intelligent agricultural decision support system that leverages multiple algorithms to provide personalized crop recommendations. The system combines historical agricultural data, real-time market prices, weather patterns, and farmer-specific constraints to generate optimal crop suggestions.

üå± 2. Crop Recommendation Algorithm

2.1 Algorithm Overview

The main recommendCrops() function is the heart of the system. It processes historical data, applies filtering, calculates economic viability, and ranks crops using a sophisticated multi-factor scoring system.

Input Parameters

Parameter Type Description
cropData CropData[] Historical crop performance records (10,000+ entries)
profile.state string Farmer's geographical state
profile.acreage number Land size in acres
profile.soilType string Soil classification (Sandy, Clay, Loamy, etc.)
profile.budget number Available investment budget (INR)
profile.season string Planting season (Kharif, Rabi, Summer)
livePrices Record Real-time market prices from AGMARKNET

2.2 Algorithm Flow

function recommendCrops(cropData, profile, livePrices) { // Step 1: Multi-level filtering filtered = applyMultiLevelFiltering(cropData, profile) // Step 2: Aggregate by crop name cropMap = groupByCropName(filtered) // Step 3: Calculate economics for each crop for each crop in cropMap { avgYield = calculateMedianYield(crop) costs = calculateCosts(crop, profile.acreage) revenue = calculateRevenue(avgYield, livePrices, profile.acreage) profit = revenue - costs score = calculateMultiFactorScore(costs, profit, yield, budget) recommendations.push(crop) } // Step 4: Sort and filter by budget constraints result = recommendations .filter(r => r.cost <= budget * 1.5) .sort((a, b) => b.score - a.score) // Step 5: Handle multiple crops if requested if (profile.multipleCrops) { result = getMultipleCropsRecommendation(result, profile) } return result }

üìä 3. Multi-Factor Scoring System

3.1 Score Components

The system uses a weighted multi-factor scoring algorithm that considers four critical aspects:

Scoring Formula

Final Score = (Budget Score √ó 0.3) + (Profit Score √ó 0.3) + (Yield Score √ó 0.2) + (ROI Score √ó 0.2) Where: ‚Ä¢ Budget Score = cost ‚â§ budget ? 1 : budget / cost ‚Ä¢ Profit Score = min(profit / 200000, 1) ‚Ä¢ Yield Score = min(avgYield / 10, 1) ‚Ä¢ ROI Score = min(profit / cost, 1)

3.2 Individual Score Calculations

Budget Score (30% weight): Measures affordability. Returns 1.0 if crop is within budget, otherwise penalizes proportionally.
Profit Score (30% weight): Normalizes profit potential with ‚Çπ200,000 as baseline. Higher profits get better scores.
Yield Score (20% weight): Evaluates productivity. Normalized with 10 tonnes/hectare as reference.
ROI Score (20% weight): Return on Investment ratio. Measures profitability efficiency.

Best for: Long-term soil health, climate resilience, government scheme eligibility

‚ö†Ô∏è 5. Risk Assessment & Crops to Avoid Algorithm

5.1 Algorithm Purpose

The identifyCropsToAvoid() algorithm performs comprehensive risk assessment to identify crops that farmers should avoid based on multiple risk factors.

5.2 Risk Factors Analyzed

Soil Incompatibility: Checks if crop is unsuitable for farmer's soil type using predefined mappings
Season Mismatch: Identifies crops planted in wrong season (e.g., Rabi crop in Kharif season)
Low Market Price: Flags crops with historically volatile or low prices
High Input Costs: Identifies crops that exceed farmer's budget capacity
Weather Sensitivity: Marks crops vulnerable to weather variability
Low Yield: Flags crops with yields below 1.5 tonnes/hectare
Negative Profit: Identifies crops where costs exceed expected revenue
Oversupply Warning: Marks crops with market saturation risk

5.3 Risk Scoring Algorithm

function identifyCropsToAvoid(cropData, profile, livePrices) { cropsToAvoid = [] for each crop { reasons = [] estimatedLoss = 0 // Check soil incompatibility if (isSoilIncompatible(crop, profile.soilType)) { reasons.push("unsuitableSoil") estimatedLoss += 15000 * acreage } // Check season mismatch if (isSeasonWrong(crop, profile.season)) { reasons.push("unsuitableSeason") estimatedLoss += 20000 * acreage } // Check market price volatility if (isLowPriceCrop(crop)) { reasons.push("lowMarketPrice") reasons.push("oversupplyWarning") estimatedLoss += 10000 * acreage } // Check high input costs vs budget if (estimatedCost > budget * 0.8) { reasons.push("highInputCost") estimatedLoss += estimatedCost - budget } // Check weather sensitivity if (isWeatherSensitive(crop)) { reasons.push("weatherRisk") estimatedLoss += 8000 * acreage } // Check low yield if (avgYield < 1.5) { reasons.push("lowYield") estimatedLoss += 12000 * acreage } // Check profit margin if (revenue < cost) { reasons.push("negativeProfitMargin") estimatedLoss = max(estimatedLoss, cost - revenue) } // Add to avoid list if significant risk if (reasons.length >= 2 || hasNegativeProfit(reasons)) { riskLevel = reasons.length >= 3 ? "high" : "medium" cropsToAvoid.push({crop, reasons, riskLevel, estimatedLoss}) } } return cropsToAvoid .sort((a, b) => b.estimatedLoss - a.estimatedLoss) .slice(0, 6) }

6.2 DanfoJS Data Preprocessing

The preprocess() function uses DanfoJS for advanced data manipulation:

Normalization Process

function preprocess(cropDf, soilDf) { // Step 1: Normalize numeric columns using Min-Max scaling numericCols = ["Area", "Production", "Rainfall", "Fertilizer", "Yield", "Price"] for each col in numericCols { min = col.min() max = col.max() normalized = (value - min) / (max - min) cropDf.addColumn(col + "_norm", normalized) } // Step 2: Explode suitable crops (one-to-many relationship) exploded = [] for each row in soilDf { crops = row.suitableCrops.split(",") for each crop in crops { exploded.push({soilType, soilQuality, crop, fertRange}) } } // Step 3: Merge datasets on crop name merged = merge(cropDf, soilExploded, on: "Crop") // Step 4: Calculate composite scores soilScore = 0.5 // Base soil compatibility marketScore = 0.5 * production_norm + 0.4 * yield_norm + 0.1 * price_norm merged.addColumn("Soil_Score", soilScore) merged.addColumn("Market_Score", marketScore) return merged }

6.3 Min-Max Normalization Formula

Normalized Value = (X - X_min) / (X_max - X_min) Where: ‚Ä¢ X = Current value ‚Ä¢ X_min = Minimum value in the dataset ‚Ä¢ X_max = Maximum value in the dataset ‚Ä¢ Result: Value in range [0, 1]

function getMultipleCropsRecommendation(recommendations, profile) { selectedCrops = [] usedCrops = new Set() // Step 1: Add the top-scoring crop if (recommendations.length > 0) { selectedCrops.push(recommendations[0]) usedCrops.add(recommendations[0].name) } // Step 2: Find companion crops for each crop in recommendations { if (selectedCrops.length >= 6) break if (usedCrops.has(crop.name)) continue isCompanion = false // Check forward compatibility for each selected in selectedCrops { companions = companionCrops[selected.name] || [] if (companions.includes(crop.name)) { isCompanion = true break } } // Check reverse compatibility cropCompanions = companionCrops[crop.name] || [] for each selected in selectedCrops { if (cropCompanions.includes(selected.name)) { isCompanion = true break } } // Add if compatible or if we need more variety if (isCompanion || selectedCrops.length < 3) { if (isCompanion) { crop.score *= 1.1 // Boost score by 10% crop.farmingType = "Intercropping Compatible" } selectedCrops.push(crop) usedCrops.add(crop.name) } } // Step 3: Fill remaining slots with high-scoring crops for each crop in recommendations { if (selectedCrops.length >= 8) break if (!usedCrops.has(crop.name)) { selectedCrops.push(crop) } } return selectedCrops }

üîç 9. Multi-Level Filtering Strategy

9.1 Cascading Filter Mechanism

The system implements a progressive fallback strategy to ensure recommendations are always available, even with incomplete or mismatched data.

Filter Levels

Level 1: State + Season Match (Strictest)
filtered = cropData.filter(c => c.state == profile.state && (c.season == profile.season || c.season == "Whole Year") )

Success Rate: ~60% | Precision: Highest

Level 2: State Match Only
if (filtered.length == 0) { filtered = cropData.filter(c => c.state == profile.state) }

Success Rate: ~85% | Precision: High

Level 3: Partial State Match (Fuzzy)
if (filtered.length == 0) { filtered = cropData.filter(c => c.state.includes(profile.state) || profile.state.includes(c.state) ) }

Success Rate: ~92% | Precision: Medium

Level 4: Season Match Only
if (filtered.length == 0 && profile.season) { filtered = cropData.filter(c => c.season == profile.season) }

Success Rate: ~95% | Precision: Medium

Level 5: Nationwide Fallback (Least Restrictive)
if (filtered.length == 0) { filtered = cropData // Use all available data }

Success Rate: 100% | Precision: Lower but ensures results

9.2 Budget Flexibility Strategy

The system also implements flexible budget constraints:

// Primary filter: Within 1.5x budget result = recommendations.filter(r => r.cost <= budget * 1.5) // Fallback 1: Within 3x budget if insufficient results if (result.length < 5) { result = recommendations.filter(r => r.cost <= budget * 3) } // Fallback 2: Best ROI crops regardless of budget if (result.length < 3) { result = recommendations.sort((a, b) => b.score - a.score) }
